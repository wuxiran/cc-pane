# 14. Resume：恢复历史会话

> 昨天那个跑到一半的 Claude / Codex，今天能**接着跑**，不用从头再来。

## 解决什么

- 续上之前的对话历史和分析，不重复“读一遍代码、理一遍思路”
- 省 context、省时间

## 在哪找

点左侧 ActivityBar 的 **最近启动**（历史图标）视图，里面列着你的启动记录，每条都可以**恢复（Resume）**。也可以直接对 Claude 说“恢复之前那个会话”，它会帮你从历史里找。

## Resume 和"新建"有什么区别

| | Resume 恢复 | 新建 |
| --- | --- | --- |
| 上下文 | 带着上次的对话历史和环境 | 干净、空白 |
| 适合 | 接着上次干 | 全新任务 |

## 什么时候能恢复

- 会话**还活着**（没退出）→ 能恢复，接着聊
- 会话**已经退出**了 → 恢复不了，只能新建一个

## 注意

- 恢复时要用**对应的 CLI 和运行环境**：上次是 Codex + WSL，就照原样恢复，别换成 Claude + 本机，否则接不上。CC-Panes 从历史记录里会带着这些信息，一般不用你操心。
- 想确认某个会话还在不在，可以在“最近启动”里看它的状态。
