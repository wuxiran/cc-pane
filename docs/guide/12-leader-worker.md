# 12. Leader / Worker 编排

> 上一篇讲"并行做什么"，这一篇讲几个实例之间**怎么协调**。CC-Panes 用的是 **指挥 - 工人** 模型。

## 一句话

- **指挥（Leader）**：你正在对话的这个主 Claude，负责拆活、派活、收结果。
- **工人（Worker）**：被它开出去的 Claude / Codex 实例，各自闷头干一块。
- **关键好处**：工人干完会**自动回报**给指挥，指挥不用一直追着问“好了没”——省心也省 token。

## 怎么发起

你不用专门说"用 leader-worker"。当你说“并行跑”“派给 Codex 几个任务”时，Claude 自动就用这套机制。

## 背后怎么协调（通俗版）

1. 指挥先把**自己**登记为 leader。
2. 每开一个工人，就把工人也**登记**到自己名下，并在交给工人的任务末尾附一句：“干完回报一声”。
3. 工人完成时，**主窗口会自动收到一行回报**（谁完成了、结果摘要）。
4. 万一回报来的时候指挥正忙、错过了，它**事后会主动核对一遍**所有工人的最终状态——所以不会漏掉谁。

> 这套“自动回报 + 事后核对”的双保险，是 CC-Panes 让多实例协作可靠的地方：即使中途有通知没接住，最终状态也能对齐。

## 你看到什么、要做什么

- **看到**：多个工人标签页、各自实时输出；有工人完成时主窗口冒出的回报通知。
- **要做**：基本只需**发起**和**收尾确认**。个别工人卡住时，你可以手动去那个窗口看看、必要时让指挥重发。

## 注意

- 这是**自动**的协调层，你不必关心内部细节；了解“指挥派活、工人自动回报、指挥事后兜底核对”这条主线即可。
- 如果某个工人长时间没动静，Claude 会按超时策略提示你是否重发。
