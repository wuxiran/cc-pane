# 用 MCP 让 AI 自己操控 CC-Panes

> 这是 CC-Panes 的**灵魂**：它不只给你用，也给 AI 用。
> CC-Panes 内置一套 orchestrator MCP，把几乎所有操作都做成了工具——于是跑在里面的 Claude / Codex 能**自己操控 CC-Panes**。

## 一句话理念

在普通终端里，AI 只能在**当前这一个窗口**里敲命令。

在 CC-Panes 里，AI 可以**跳出单个窗口**，像你一样去操作整个工作台：开新实例、读别的会话的输出、建工作空间、把活派给另一个 AI、记笔记…… 因为这些操作 CC-Panes 全都暴露成了 **MCP 工具**。

举个例子，你只要说：

> “把这三个互相独立的任务并行跑了，完成后汇总给我。”

它就能自己完成：开三个 Claude / Codex 实例 → 分别派任务 → 盯着各自状态 → 都跑完后收集结果汇报给你。**整个过程你不用手点一次。**

## 它能做哪些（几乎覆盖你界面上的所有操作）

| 类别 | AI 能做的事 | 代表能力 |
| --- | --- | --- |
| **启动与监控会话** | 开新实例、读它的实时输出、给它发指令、结束它 | 启动任务 / 查会话状态 / 取会话输出 / 提交输入 |
| **工作空间与项目** | 扫描目录、批量导入仓库、新建工作空间、加项目 | 扫描目录 / 建工作空间 / 加项目 |
| **恢复历史会话** | 找到上次的会话并接着跑 | 查启动历史 → 按 resumeId 启动 |
| **多实例编排** | 一个“指挥”实例把活派给多个“工人”实例，自动回报 | 注册 leader / worker、上报结果 |
| **布局分流** | 把新任务开到指定布局 / 面板 | 查面板 → 指定 layout / pane 启动 |
| **记忆与待办** | 把决策、经验写进共享记忆，管理 Todo | 记忆增删查 / Todo 增删查 |
| **桌面交互** | 打开文件、发通知、让桌面宠物 cc酱 说话 | 打开文件 / 触发通知 / cc酱说话 |

> 完整工具有几十个，这里只列了最常用的几类。要点是：**你在界面上能做的，AI 基本都能通过 MCP 做。**

## 你不用配，它已经在那

当你用 CC-Panes 启动一个 Claude / Codex 时，CC-Panes 会**自动把这套 orchestrator MCP 注入**进去——所以实例一跑起来，AI 就已经“长出了操控 CC-Panes 的手”，不需要你额外配置。

如果你还想接入别的 MCP server（比如某个第三方工具），可以到 **设置 → 共享 MCP** 里统一管理。

## 配套的现成玩法（Skills）

CC-Panes 还基于这套 MCP 封装了一批开箱即用的 skill，你直接用就行：

- **并行跑任务**：把大任务拆成多个子任务，开多个实例同时跑、自动汇总
- **Plan → Codex**：Claude 负责规划，把实现派给 Codex
- **Plan 同行评审**：另开一个 Codex 从不同角度审你的方案，专治“自己审自己看不出问题”
- **分派 Todo**：把待办列表批量分发给新实例
- **Fork 会话**：带着当前对话的上下文摘要，另开一个实例继续探索

这些玩法的详细步骤，见本章其余各篇。

## 为什么这件事重要

普通的多终端工具，AI 是“被关在格子里”的——你得当“人肉调度员”，手动开窗口、复制粘贴、来回盯。

CC-Panes 把工作台本身变成了**AI 可编程的对象**：你只需要描述目标，AI 自己调度。这就是它从“分屏终端”跃迁为“**AI 编排工作台**”的关键。
