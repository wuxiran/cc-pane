## 1. 陈旧 busy 回落不得放行自动注入(docs/50 严重 1)

- 把"展示降级"与"允许自动提交"分离:查询侧回落(10 分钟无 hook/OSC)只允许影响 UI 状态灯/stats 展示;**补投电平扫描的放行条件收紧**为:状态机显式 Idle/WaitingInput(真实跃迁,非回落),或回落 + **PTY 输出同样静默**(`last_output_at` 也超过阈值,说明前台确实无进程在跑)。
- 实现建议:状态查询增加"回落来源"标记(或提供带/不带回落两个查询),电平扫描用严格版;`terminal_commands.rs` 的 UI 查询用展示版。
- 测试:hook 静默 + PTY 持续输出 → 扫描不投;hook 静默 + PTY 同样静默 → 投;真实 WaitingInput → 投。

## 2. 项目身份 key 与持久化路径分离(docs/50 严重 2)

- `project_identity` 的 canonical 只作**去重/等价比较 key**,**不再改写持久化 path**:add/迁移保留用户提供且宿主可用的原路径(添加时已验证存在的那个形式);去重比较走 canonical key。
- 已被 0.11.0 迁移改写过的数据:迁移逻辑加修复路径——持久化 path 在当前宿主不可用而其等价形式可用时,回写为可用形式(幂等,保 .bak 惯例)。
- 测试:Unix 宿主注册 `/mnt/d/repo` → 持久化仍为 `/mnt/d/repo` 且可启动;Windows 注册三形式仍去重为一条;跨形式 launch_task 校验仍放行;修复迁移把坏数据改回可用形式。

## 3. debounce 溢出不得丢删除语义(docs/50 严重 3)

- 128 路径溢出时不再整批静默跳过:溢出批**保留其中的删除事件**(Removed 单独收集不设 128 上限,或上限独立且更高),修改类事件可丢(下次写入自然补的语义只对它们成立);若实现上保留删除不可行,则溢出时给该项目标 dirty 并调度一次有界、可取消的删除对账(对 repo 已知文件做存在性核对,生成 tombstone)。
- 30k 通道整批丢弃同理复核:丢弃批含删除时同样标 dirty。
- 测试:一次删除 200 个文件 → 全部产生 tombstone;溢出批中的修改事件丢弃仍可接受;对账可取消且有界。

## 4. NSIS $INSTDIR 撇号转义(docs/50 建议,顺手)

- `installer-hooks.nsh` 不把 `$INSTDIR` 内插进 PowerShell 单引号源码:改环境变量/参数传递,或做单引号转义('→'');保持 IfSilent 语义不变。
