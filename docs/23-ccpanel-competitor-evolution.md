要点：

- **信息架构**：标题栏面包屑（工作区 `1-app` / 项目 `AI`，均可下拉切换）→ 左侧栏两段式（上：项目池；下：按项目分组的会话列表）→ 主区（会话/文件双 Tab + 终端标签）。
- 左侧栏**项目区和会话区是分离的**：项目区是"可启动的目录池"（带路径），会话区是"运行中/历史实例"，按项目名分组、组头带数量徽标 `(1)`、可折叠。
- **双状态栏**：TUI 内部 statusline（powerline 药丸风格，含模型[上下文规格]、Ctx 用量+百分比、项目、git 分支）+ 应用底部状态栏（应用自绘）。两者数据独立，前者疑似注入 Claude Code statusline 配置实现。
- 网格模式是**独立的顶层视图**（替换整个主区，含自己的分组 Tab 行），不是会话页里的分屏。

### 2.1 标题栏（自定义无边框）

| 控件 | 说明 |

| ➕ 新建 | 快速新建（文件图标+） |
| 👥 成员/多实例 | 疑似多 Agent/协作入口 |
| 💡 提示 | 使用技巧 |

### 2.2 左侧栏

- `＋ 新建终端` 大按钮 + 旁边一个 ✦ 小方按钮（快捷启动/AI 推荐启动）。
- `🔍 筛选终端…` —— 过滤**会话**。
- 项目分组头：`▼ 项目 (4)`，右侧 hover 图标：🗑（清空项目列表）、📁（添加项目——弹**系统目录选择器**，见 f_010/f_011，选择目录即入池）。
- 第二个搜索框 `🔍 搜索项目…` —— 单独过滤**项目**（项目多时两级搜索并存）。
- 项目行：文件夹图标 + 项目名 + 完整路径（次级灰字）。
- 会话分组：按项目名分组 `▼ Administrator (1)`；会话卡片包含：
  - `CC` 圆角徽标（CLI 类型，Claude Code 橙色）
  - 会话名 = **首条 prompt 截断**（如 "Syntax theme: Mon…"）或目录名
  - 状态字：`空闲`（橙字）/ `运行中`（绿字 + 行首绿点）
  - cwd 路径（灰字小号）+ 相对时间（`2分钟前`/`刚刚`，右对齐）
  - 选中态：整卡橙色描边 + 浅橙底

### 2.3 主区（会话页）

- 顶层 Tab：`💬 会话` / `📄 文件`（文件浏览器与终端平级，同一项目上下文）。
- 终端标签行：每个标签带**状态圆点**（橙=空闲，随状态变色）+ 标题截断 + `＋` 新建；标签溢出时出现 `>` 滚动按钮。
- 右侧常驻：`💬 Chat` 按钮（聊天模式）+ `⊞ 网格` 按钮（进入网格视图）。
- 终端区：xterm 渲染 Claude Code TUI；空闲时输入框有占位建议 `Try "fix typecheck errors"`；右下角悬浮当前 effort 指示 `● high · /effort`。

## 3. 功能模块详解

### 3.1 快速启动弹窗「启动终端」（f_012~f_014, f_019）

从项目行触发，小弹窗：

- 头部：📂 项目名 + 完整路径（只读确认）
- **CLI 类型**：`Claude Code` / `OpenCode` / `Codex` / `Grok` 四个 segment 按钮
- **努力程度**：`默认 / 低 / 中 / 高 / 极高 / 最大` 六档 segment（选择后下方 Args 行实时变化）
- **Args 预览行**：灰字实时显示将要附加的 CLI 参数（如 `--dangerously-skip-permissions`）——参数透明化
- **API 供应商**：两级下拉——Provider（`MiniMax-M3 (copy) (默认)`）+ 模型（`MiniMax-M3`）
- ☐ `⊞ 创建后添加到网格`
- `取消` / `▷ 创建`

### 3.2 完整新建终端弹窗（f_009，从侧栏大按钮触发）

大弹窗，比快速启动多出一整层：

- **CLI 类型**：多一个 `Plain Shell`（纯终端，不挂 CLI）；右上角 **Mode 切换：`>_ Terminal` / `💬 Chat`**
- **模板**（7 个带 emoji 图标的卡片）：`✨默认` / `🧠Karpathy 编程` / `💻专业编程` / `📄文档写作` / `📊PPT/演示` / `🏗架构设计` / `📈数据分析` —— 场景化预设（system prompt + 参数组合）
- **会话启动注入**（三选一 segment）：`不注入` / `本地文件` / `Skill 库` —— 启动时把文件内容或选中 Skill 注入为首条上下文
- **工作配置**：
  - 工作目录 * ：三种选取方式 Tab —— `目录`（路径输入+📁浏览）/ `工作区`（从工作区列表选）/ `运行中 1`（复用运行中会话的目录）
  - 昵称：占位 `例如：我的项目、后端 API`
  - **独立工作树** toggle：副标题"在独立 git 工作树中运行" —— worktree 一键化
- **AI 配置**：
  - API 提供商下拉（带 `sk-***` 掩码显示）+ 右上 `⚙ 管理` 直达 Provider 管理器
  - 模型下拉
  - **CLI 参数 chips**（点亮即启用，选中橙色）：`跳过权限` / `限制工具` / `禁用工具` / `最大轮数` / `追加提示` / `详细日志` / `MCP 配置`
  - `其他自定义参数…` 自由输入框
- ☐ `⊞ 创建后加入网格`；`取消` / `创建`

### 3.3 网格视图（f_022~f_029）

- 顶部第一行：**网格分组 Tab**（`网格1` + 数量徽标）+ `＋` 新建分组 —— 多套网格并存
- 第二行工具条：
  - 左：`⊞ 网格视图 (2/8)` —— 当前格子占用/总数
  - 中右：**10 个布局预设图标**：单格 / 左右两列 / 三列 / 四列 / 上下两行 / 三行 / 横向堆叠 / 3×3 / 2×2 / 2×3 —— 点击即时重排
  - `📚 全部添加 ▾` 下拉（f_023）：`全部 (2)` 一键铺入所有会话；下方 `添加项目` 分组列出每个项目（`Administrator 1`、`AI 1`）可按项目批量添加
  - `⤡ 退出网格` 返回会话页
- 每格结构：
  - 迷你标题栏：`CC` 徽标 + 会话名 + **8 个操作图标**（📄复制缓冲 / 📂打开目录 / ⧉克隆终端 / 🧹清空 / ⇱移回会话页 / ⤢全屏放大 / ✕关闭，另有一个纸张图标疑似导出）
  - 终端主体（完整 xterm，含 TUI statusline）
  - **迷你状态栏**：⏱耗时 / 上下文进度条 / 状态点 / 🔌📋🔄 / ≡任务队列 —— 与主状态栏同构
- 空格子：居中大 `＋`，文案"网格中无终端 / 点击任意 + 单元格添加终端"
- 布局切换动画流畅，格子内容 keep-alive（切布局不重建终端）

### 3.4 API Provider 管理器（f_006/f_007，重量级）

- 顶部**按 CLI 分 Tab**：`Claude (5)` / `OpenCode (1)` / `Codex (3)` / `Grok (0)` —— 每种 CLI 独立 Provider 池（各自参数 schema 不同）
- 右上角图标组：`⬆导出` / `⬇导入` / `📋复制配置` / `📋⁺粘贴配置` / `🌐打开官网` / `✕`
- 左列（Provider 列表）：
  - 🔍 搜索框
  - Provider 卡：名称 + Base URL + `sk-***` 掩码 + 模型数（`minimax · 3 models`）；默认者带 ⭐
  - hover 出现行内操作：`↑` `↓` 排序 / `⊘` 禁用 / `⧉` 克隆 / `🗑` 删除
  - 底部 `＋ 新建提供商`
- 右侧（Provider 编辑区）：
  - **快速模板 chips**（OpenCode Tab 下）：`OpenAI` / `DeepSeek` / `智谱 GLM` / `MiniMax` / `通义千问` / `Kimi` / `硅基流动` / `Anthropic` / `Google Gemini` —— 点击预填 Base URL / NPM 包名
  - 字段：`Provider Key`（内部标识）/ `提供商名称` / `NPM Package`（OpenCode 特有，如 `@ai-sdk/openai-compatible`，下拉）/ `Base URL` / `API Key`（👁 可见性切换）
  - **模型列表**（Claude Tab 下，f_006）：标题旁说明"可添加多个模型，各自维护上下文窗口"；每行 = 模型名输入框 + **上下文窗口下拉（1M）** + `↑↓` 排序 + `⭐` 设默认（默认行高亮黄框 + `★默认` 徽标）+ `🗑` 删除
  - `保存提供商` / `🗑 删除`
- 观察到的真实配置：一个 Provider 下挂 `claude-sonnet-4-6 / claude-opus-4-8 / claude-opus-4-7 / claude-opus-4-6 / claude-fable-5(默认)` 全系列，各 1M 窗口 —— **中转站多模型场景是一等公民**

### 3.7 通知与任务（f_022）

- 侧栏状态实时跃迁：空闲(橙) ↔ 运行中(绿+绿点)，标签页圆点同步变色

### 3.8 其他观察

- 添加项目 = 直接调**系统目录选择器**（f_010/f_011），选完即入项目池——没有"扫描目录批量导入"的中间页（我们有 scan_directory，更强）
- `文件` Tab：项目文件浏览器与终端平级
- Chat 模式：同一会话的聊天 UI（视频未深入展示，仅见入口）
- 语音输入：PRO 功能，`Alt+X` 全局快捷键
- 权限管家（Agent Guardian 分组）：集中式权限放行策略管理
- TUI 欢迎框内可见 `MiniMax-M3[1m] with high effo… · API Usage Billing` —— 启动参数（模型+effort+计费模式）直接体现在 Claude Code 启动横幅

### 明确不做 / 不抄

1. **文本猜测会话状态**——CCPanel 状态机制不明；我们坚持 OSC/hook 双通道（Known Gotchas 明令禁止回退）。
2. **语音输入**——Windows WebView 录音权限与收益不成比例，观望。
3. **PRO 付费墙拆分**——阶段不符。
4. **双向 Chat 模式**——同一 PTY 的聊天化双向交互成本极高，只做 P2-8 只读视图。
5. **放弃 scan_directory 换系统选择器**——我们的批量扫描导入比它的单目录选择强，保留。
